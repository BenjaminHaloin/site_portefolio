DressCode

Projet vitrine immersif réalisé pour un contexte RP. Une première preuve concrète de structure, contenu et identité visuelle, avec un site publié en ligne.

Présentation du projet

DressCode est un site vitrine imaginé pour présenter une entreprise fictive dans un univers RP. L’objectif était de créer une page immersive, lisible et cohérente, avec une ambiance visuelle propre au projet.

Accès au projet

DressCode est consultable en ligne via Netlify. Le dépôt GitHub permet aussi de voir la structure du projet, les fichiers HTML/CSS et l’organisation du code.

Voir le site en ligneVoir le code

Contexte

Le projet a été réalisé dans un contexte RP. Ce cadre m’a permis de travailler un cas concret : présenter une activité, organiser des informations et créer une ambiance visuelle autour d’un univers.

Objectifs

  • Créer un site vitrine immersif.
  • Structurer les sections principales.
  • Présenter une entreprise fictive.
  • Travailler une première identité visuelle.
  • Mettre en avant un projet terminé, publié et consultable.

Ce que le projet démontre

  • Capacité à terminer et publier un projet.
  • Organisation du contenu.
  • Première approche de la mise en page.
  • Travail d’ambiance visuelle.
  • Compréhension de la logique d’un site vitrine.

Ce que j’ai appris

DressCode m’a permis de comprendre qu’un site web ne repose pas seulement sur le code. Il faut aussi penser la hiérarchie des informations, la lisibilité, la cohérence visuelle et l’expérience de la personne qui visite la page.

Améliorations possibles

  • Nettoyer le HTML et le CSS.
  • Améliorer le responsive mobile.
  • Ajouter des captures du rendu.
  • Rendre le README plus professionnel.
  • Améliorer l’accessibilité.
